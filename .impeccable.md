# Impeccable Design Guide — Reki 歴

## Design Context

### Users
Power learners who treat their tools like instruments — people who care about efficiency, customization, and clean design. They use Reki in two modes: fast-paced daily review (5-15 min, rhythm matters) and deep sessions for organizing decks, creating cards, and studying complex material (30+ min, comfort matters). These are the same people who use Obsidian, Neovim, or Linear — they expect a tool that respects their time and intelligence.

### Brand Personality
**Precise. Quiet. Sharp.**

Reki feels like a well-tuned instrument — not flashy, not cold, just exactly right. It doesn't try to motivate or gamify. It trusts you to show up. The interface recedes so the content can breathe. Every pixel is deliberate; nothing decorative survives without a functional reason.

Emotional goals: confidence in the tool, calm focus during review, satisfaction from a clean workflow.

### Aesthetic Direction

**Visual tone:** Dark-first knowledge tool. Dense but not cramped — information-rich surfaces with clear hierarchy. Think Obsidian's editor panes meets Linear's navigation precision.

**Key references:**
- **Obsidian** — Dark palette, customizable, respects power users, knowledge-tool DNA
- **Linear** — Navigation precision, clean data density, professional polish
- **Raycast** — Speed-oriented UI, keyboard-first, minimal chrome
- **Warp** — Modern terminal aesthetic, functional beauty

**Anti-references:**
- Notion — Too canvas-y, too cute, too many rounded corners
- Duolingo — Gamified, cartoon-y, patronizing for serious learners
- Old Anki — Cluttered, no hierarchy, 2005 desktop energy
- Any glassmorphism, gradient backgrounds, or decorative blobs

**Theme:** Follow system preference. Both dark and light modes are first-class citizens. Dark mode is the design-first target.

### Design Principles

1. **Content is king** — The card content area owns the viewport. UI chrome (sidebar, toolbars, status) stays peripheral and quiet. Full-width content area for review, not centered small cards.

2. **Keyboard-native** — Every action reachable by keyboard. Visual affordances for mouse users, but the fast path is always keys. Anki's 1/2/3/4 rating shortcuts are muscle memory — honor that.

3. **Hierarchy through typography, not decoration** — Size, weight, and spacing create structure. No colored icons in circles, no decorative dividers, no ornamental elements. If you need to distinguish something, use type scale or opacity.

4. **Restrained color** — The UI is essentially monochrome. Color appears only for semantic meaning: ratings (again/hard/good/easy), states (success/warning/error), and the rare accent for active/selected elements. Never decorative.

5. **Density with clarity** — Pack information tightly but never confusingly. Use whitespace strategically — generous within content, tight in navigation. The browse view can be dense; the review view should breathe.

## Typography

**Display:** A geometric sans-serif with personality — something sharper than Inter but not experimental. Weight 600-700 for headings, 500 for sub-headings.

**Body:** A screen-optimized sans-serif with good tabular numerals. Weight 400 for reading, 500 for emphasis. Must render crisp at 13-15px.

**Monospace:** Matching monospace family for code blocks, cloze deletions, and data. Must align well with the body font.

**Scale (rem-based, 15px base):**
- xs: 0.733rem (11px)
- sm: 0.867rem (13px)
- base: 1rem (15px)
- lg: 1.2rem (18px)
- xl: 1.467rem (22px)
- 2xl: 1.867rem (28px)
- 3xl: 2.4rem (36px)

## Color Strategy

### Dark Mode (design-first target)

**Surfaces** — A neutral-cool gray ramp. Not blue-tinted (avoid GitHub dark), not warm (avoid sepia). True neutral or very slight cool lean.
- App background: ~#111114
- Sidebar/panels: ~#161619
- Surface (cards, modals): ~#1A1A1E
- Elevated (dropdowns, popovers): ~#222226
- Hover: ~#2A2A2E

**Borders** — Subtle, ~8-12% opacity white or a dedicated gray. Never prominent.
- Default: ~#2A2A2E
- Strong: ~#3A3A3E

**Text** — High contrast primary, stepping down through secondary/muted/ghost.
- Primary: ~#E8E8EC (not pure white — too harsh)
- Secondary: ~#9898A0
- Muted: ~#5E5E66
- Ghost: ~#3E3E44

**Accent** — A single muted accent for active states. Cool-toned, unsaturated. Think Obsidian's purple or a slate blue. Used only for: active nav item, focused input border, selected state.
- Accent: ~#7B8AF5 (muted periwinkle) or ~#8B8BF5 (muted lavender)
- Accent text: same as accent or slightly brighter

**Rating buttons (the only vivid color in the UI):**
- Again: #C45C5C (muted red — clear "wrong" signal)
- Hard: #C49A4A (muted amber — friction signal)
- Good: #5CA06A (muted green — positive signal)
- Easy: #5A80B0 (muted blue — confident signal)

### Light Mode

Invert the surface ramp. Keep rating button hues but reduce saturation ~10%.
- App background: #F8F8FA
- Sidebar: #F0F0F4
- Surface: #FFFFFF
- Text primary: #1A1A1E
- Text secondary: #6B6B72
- Border: #E2E2E6
- Accent: #5B6AD5 (slightly darker than dark mode accent for contrast)

### Semantic Colors
- Success = Good rating color
- Warning = Hard rating color
- Error = Again rating color
- Info = Easy rating color

## Spacing

Base unit: 4px. Scale:
- 2xs: 2px
- xs: 4px
- sm: 8px
- md: 16px
- lg: 24px
- xl: 32px
- 2xl: 48px
- 3xl: 64px

Density: comfortable in review mode (content breathes), compact in browse/list mode (data density).

## Layout

- **Structure:** Collapsible sidebar (200-240px) + full-width main content
- **Review mode:** Card content fills the main area. Answer buttons pinned to bottom or below content.
- **Browse mode:** Table/list with virtual scroll. Dense rows, sortable columns.
- **Editor mode:** Split or tabbed — edit on one side, preview on the other.
- **Max content width:** No artificial max-width in review (respect full-width choice). Browse tables use full width. Editor may constrain prose to ~720px for readability.

## Border Radius

Small and consistent. Nothing pill-shaped.
- sm: 4px (tags, inline badges, code)
- md: 6px (buttons, inputs, nav items)
- lg: 8px (cards, panels, modals)

## Motion

Functional only. Nothing decorative.
- Easing: cubic-bezier(0.16, 1, 0.3, 1)
- Micro: 80ms (hover, focus ring)
- Short: 150ms (button press, toggle, nav highlight)
- Medium: 250ms (panel slide, card flip, modal)
- What animates: hover states, card flip, sidebar collapse, modal open/close
- What doesn't: page navigation, content load, tab switch — all instant
- Respect `prefers-reduced-motion`: disable card flip and panel animations

## Iconography

Minimal geometric symbols. No emoji, no filled icons, no colored icons.
- Style: 1.5px stroke, 16-20px, monochrome
- Source: Lucide or Phosphor (thin variant)
- Usage: navigation, actions, status indicators only

## Component Notes

**Sidebar:** Dark, quiet, collapsible. Deck tree with expand/collapse. Active item uses accent color or brighter text — no background highlight pill.

**Review card:** Content fills available space. Question on show, answer revealed below on flip. No visible card border in review — the content IS the card.

**Rating bar:** 4 buttons (Again / Hard / Good / Easy) with keyboard shortcuts prominently shown. Interval preview on hover. Buttons use the rating colors as background with dark text.

**Browse/search:** Filterable table with virtual scroll. Sortable columns. Inline editing. Quick preview panel (right side or bottom).

**Editor:** Markdown-native with live preview. Support for cloze `{{c1::deletions}}`, images, LaTeX (KaTeX), code blocks.
